\begin{frame}
  \frametitle{Public clouds}
  
  We've looked at an example of a \emph{public cloud} service, AWS. Running operations on a public cloud is often an east and cost effective way to provide networked services.
  
  \end{frame}

\begin{frame}
  \frametitle{Three main components}
  
  Recall the three main components of a cloud service:
  
  \begin{itemize}
  \item Virtual \textbf{compute} service
  \item Virtual \textbf{storage} service
  \item Virtual \textbf{networking} service
  \end{itemize}
  
  Along with these core services, a cloud service will typically implement other supporting services.
  
\end{frame}

\begin{frame}
  \frametitle{Cloud interfaces}
  
  Recall also that AWS provided three different interfaces:
  
  \begin{itemize}
   \item A graphical web interface
   \item A command-line interface
   \item A remote API
   \end{itemize}
   
   A private cloud interface may provide any combination of these (but probably needs at least one).
\end{frame}

\begin{frame}
  \frametitle{A big task}
  
  The big thing to take away from all this is that building and running a private cloud is a \textbf{big} task. Do not choose to do so lightly.
  \end{frame}

\begin{frame}
  \frametitle{OpenStack}
  
  We're going to use OpenStack to run our cloud.
  
  \begin{itemize}
    \item A FOSS platform for running cloud services
    \item Really a large, modular collection of services
    \item Project launched in 2010
    \item Managed by the OpenStack Foundation
  \end{itemize}
